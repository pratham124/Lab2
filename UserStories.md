\## 2. User Stories

\### 2.1 Registration & Authentication

\- As a \*\*user\*\*, I want to register with my email and password, so that I can access CMS features.

\- As a \*\*registered user\*\*, I want to log in securely, so that I can access my dashboard.

\- As a \*\*registered user\*\*, I want to change my password, so that I can keep my account secure.

\---

\### 2.2 Paper Submission (Author)

\- As an \*\*author\*\*, I want to submit a paper with metadata (Name of authors, their affiliation and contact information, Abstract, Keywords of the paper, Main source of the paper), so that it can be reviewed.

\- As an \*\*author\*\*, I want to upload my manuscript in PDF, Word, or LaTeX format, so that it meets submission rules.

\- As an \*\*author\*\*, I want to save my submission progress, so that I can complete it later.

\- As an \*\*author\*\*, I want to receive the final decision on my paper, so that I know whether it is accepted.

\---

\### 2.3 Reviewer Assignment (Editor)

\- As an \*\*editor\*\*, I want to assign 3 reviewers to submitted papers, so that each paper is evaluated fairly.

\- As an \*\*editor\*\*, I want the system to prevent assigning more than five papers to a reviewer, so that workload is balanced.

\- As an \*\*editor\*\*, I want to be notified of assignment rule violations, so that I can correct them.

\---

\### 2.4 Paper Reviewing (Reviewer & Editor)

\- As a \*\*reviewer\*\*, I want to receive review invitations, so that I can accept or reject them.

\- As a \*\*reviewer\*\*, I want to access assigned papers, so that I can review them.

\- As a \*\*reviewer\*\*, I want to submit a completed review form, so that the editor can assess the paper.

\- As an \*\*editor\*\*, I want to view all completed reviews, so that I can make a final decision.

\- As an \*\*editor\*\*, I want to send acceptance or rejection decisions to authors, so that they are informed.

\---

\### 2.5 Scheduling (Administrator & Editor)

\- As an \*\*administrator\*\*, I want the system to generate a conference schedule, so that papers are assigned time slots.

\- As an \*\*editor\*\*, I want to edit the generated schedule, so that conflicts can be resolved.

\- As an \*\*author\*\*, I want to receive the final schedule, so that I know when to present.

\- As an \*\*attendee\*\*, I want to view the published schedule, so that I can plan my attendance.

\---

\### 2.6 Conference Registration & Payment (Attendee)

\- As an \*\*attendee\*\*, I want to view registration prices, so that I know the cost of attending.

\- As an \*\*attendee\*\*, I want to pay the registration fee online, so that I can attend the conference.

\- As an \*\*attendee\*\*, I want to receive a payment confirmation ticket, so that I have proof of registration.

\---
